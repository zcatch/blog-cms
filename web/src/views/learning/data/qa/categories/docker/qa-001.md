# Docker 容器与镜像的区别及启动方式

## ❓ 问题

运行 `docker run -d my-nginx` 时报错：`Unable to find image 'my-nginx:latest' locally`，但 `docker ps -a` 显示已有名为 `my-nginx` 的容器。为什么会出现这个错误？

## ✅ 解答

### 核心概念区分

Docker 中有两个重要概念需要区分：

| 概念 | 说明 | 示例 |
|------|------|------|
| **镜像 (Image)** | 应用程序的模板，用于创建容器 | `nginx`, `mysql`, `redis` |
| **容器 (Container)** | 镜像的运行实例，可以启动、停止 | `my-nginx`, `my-app` |

### 问题分析

```bash
# 查看已有容器
docker ps -a
# 输出：NAME 为 my-nginx，IMAGE 为 nginx

# 错误命令
docker run -d my-nginx
# Docker 将 my-nginx 当作镜像名去查找，但实际上它是容器名
```

**错误原因：**
- `docker run` 命令需要的是**镜像名**，而不是容器名
- `my-nginx` 是容器的名称，不是镜像名
- 实际的镜像名是 `nginx`

### 正确的操作方式

#### 1. 启动已存在的容器

```bash
# 启动已停止的容器
docker start my-nginx

# 查看运行状态
docker ps
```

#### 2. 创建新容器

```bash
# 使用正确的镜像名创建新容器
docker run -d --name my-nginx-2 nginx

# 或者删除旧容器后重新创建
docker rm my-nginx
docker run -d --name my-nginx nginx
```

#### 3. 带端口映射启动

```bash
# 映射端口 80 到主机 8080
docker run -d -p 8080:80 --name my-nginx nginx
```

### 常用命令对比

| 命令 | 用途 | 示例 |
|------|------|------|
| `docker run` | 从镜像创建并运行新容器 | `docker run -d nginx` |
| `docker start` | 启动已存在的容器 | `docker start my-nginx` |
| `docker stop` | 停止运行中的容器 | `docker stop my-nginx` |
| `docker restart` | 重启容器 | `docker restart my-nginx` |
| `docker rm` | 删除容器 | `docker rm my-nginx` |
| `docker rmi` | 删除镜像 | `docker rmi nginx` |

## 💡 小贴士

**查看镜像和容器：**
```bash
# 查看所有镜像
docker images

# 查看所有容器（包括停止的）
docker ps -a

# 只查看运行中的容器
docker ps
```

**容器命名规则：**
```bash
# 不指定名称，Docker 会自动生成随机名称
docker run -d nginx

# 使用 --name 指定容器名称
docker run -d --name my-custom-name nginx
```

**容器生命周期：**
1. 创建：`docker run` 或 `docker create`
2. 启动：`docker start`
3. 停止：`docker stop`
4. 删除：`docker rm`

## 🔗 相关命令

```bash
# 查看容器详细信息
docker inspect my-nginx

# 查看容器日志
docker logs my-nginx

# 进入容器内部
docker exec -it my-nginx bash

# 查看容器资源使用
docker stats my-nginx
```

## 📌 关键要点

1. **镜像是模板，容器是实例** - 一个镜像可以创建多个容器
2. **docker run 用于创建新容器** - 需要提供镜像名
3. **docker start 用于启动已有容器** - 需要提供容器名或 ID
4. **容器名和镜像名是不同的** - 不要混淆使用
