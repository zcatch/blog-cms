# Docker + Nginx 多项目部署方案（使用挂载）

## 📋 场景说明

在开发环境中，需要使用一个 Nginx 容器管理多个前端项目，通过域名区分不同项目，使用 Bind Mount（绑定挂载）方式实现代码实时更新。

---

## 🎯 方案架构

### 目录结构

```
/home/blue/
├── nginx/
│   └── conf.d/
│       ├── project1.conf
│       ├── project2.conf
│       └── vue-learning.conf
└── www/
    ├── project1/
    │   └── dist/
    ├── project2/
    │   └── dist/
    └── web_deploy/
        └── dist/
```

### 核心思路

- **一个 Nginx 容器**：管理所有项目
- **目录挂载**：宿主机目录映射到容器
- **独立配置**：每个项目一个 `.conf` 文件
- **域名区分**：通过 `server_name` 区分不同项目

---

## 🚀 部署步骤

### 1. 创建目录结构

```bash
# 创建 Nginx 配置目录
mkdir -p /home/blue/nginx/conf.d

# 创建项目目录（如果不存在）
mkdir -p /home/blue/www

# 修改目录所有者
sudo chown -R blue:blue /home/blue/nginx
```

### 2. 创建项目配置文件

为每个项目创建独立的 Nginx 配置文件：

```bash
# 示例：/home/blue/nginx/conf.d/vue-learning.conf
nano /home/blue/nginx/conf.d/vue-learning.conf
```

配置内容：

```nginx
server {
    listen 80;
    server_name web.local learning.local www.learning.local;
    
    # 项目根目录（容器内路径）
    root /usr/share/nginx/html/web_deploy/dist;
    index index.html;

    # Vue Router History 模式支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
}
```

### 3. 构建前端项目

```bash
# 进入项目目录
cd /home/blue/www/web_deploy

# 安装依赖
npm install

# 构建生产版本
npm run build

# 确认 dist 目录已生成
ls -la dist/
```

### 4. 运行 Nginx 容器

```bash
docker run -d \
  --name my-web \
  -p 8080:80 \
  -v /home/blue/www:/usr/share/nginx/html \
  -v /home/blue/nginx/conf.d:/etc/nginx/conf.d \
  --restart unless-stopped \
  nginx
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `-d` | 后台运行 |
| `--name my-web` | 容器名称 |
| `-p 8080:80` | 端口映射：宿主机 8080 → 容器 80 |
| `-v /home/blue/www:/usr/share/nginx/html` | 挂载所有项目目录 |
| `-v /home/blue/nginx/conf.d:/etc/nginx/conf.d` | 挂载配置目录 |
| `--restart unless-stopped` | 自动重启策略 |

### 5. 验证配置

```bash
# 测试 Nginx 配置语法
docker exec my-web nginx -t

# 查看加载的域名配置
docker exec my-web nginx -T | grep server_name

# 查看容器运行状态
docker ps | grep my-web

# 查看端口映射
docker port my-web
```

### 6. 配置域名访问

#### Linux/WSL 系统（服务器端）

```bash
# 编辑 hosts 文件
sudo nano /etc/hosts

# 添加域名映射
127.0.0.1  web.local
127.0.0.1  learning.local
127.0.0.1  www.learning.local

# 测试域名解析
ping web.local

# 测试 HTTP 访问
curl http://web.local:8080
```

#### Windows 系统（WSL 场景）

**步骤 1：以管理员身份打开记事本**
- 右键"记事本" → "以管理员身份运行"

**步骤 2：打开 hosts 文件**
```
文件 -> 打开 -> 输入路径：
C:\Windows\System32\drivers\etc\hosts
```
注意：文件类型选择"所有文件"

**步骤 3：添加域名映射**
```
127.0.0.1  web.local
127.0.0.1  learning.local
127.0.0.1  www.learning.local
```

**步骤 4：刷新 DNS 缓存**
```powershell
ipconfig /flushdns
```

**步骤 5：验证**
```powershell
# 测试域名解析
ping web.local

# 浏览器访问
http://web.local:8080
```

## 🔄 日常操作

### 添加新项目

```bash
# 1. 创建新项目配置
nano /home/blue/nginx/conf.d/project2.conf

# 2. 配置内容
server {
    listen 80;
    server_name project2.local;
    root /usr/share/nginx/html/project2/dist;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
}

# 3. 重载 Nginx 配置
docker exec my-web nginx -s reload

# 4. 添加 hosts 映射
# Linux: sudo nano /etc/hosts
# Windows: C:\Windows\System32\drivers\etc\hosts
127.0.0.1  project2.local
```

### 更新项目代码

```bash
# 1. 拉取最新代码
cd /home/blue/www/web_deploy
git pull

# 2. 重新构建
npm run build

# 3. 刷新浏览器即可看到更新
# 不需要重启容器！
```

### 修改 Nginx 配置

```bash
# 1. 编辑配置文件
nano /home/blue/nginx/conf.d/vue-learning.conf

# 2. 测试配置
docker exec my-web nginx -t

# 3. 重载配置（不中断服务）
docker exec my-web nginx -s reload
```

### 查看日志

```bash
# 查看容器日志
docker logs -f my-web

# 查看 Nginx 访问日志
docker exec my-web tail -f /var/log/nginx/access.log

# 查看 Nginx 错误日志
docker exec my-web tail -f /var/log/nginx/error.log
```

### 容器管理

```bash
# 停止容器
docker stop my-web

# 启动容器
docker start my-web

# 重启容器
docker restart my-web

# 删除容器
docker rm -f my-web

# 进入容器
docker exec -it my-web sh
```

## 🔍 故障排查

### 问题 1：浏览器无法访问

**检查步骤：**

```bash
# 1. 确认容器运行
docker ps | grep my-web

# 2. 确认端口映射
docker port my-web
# 应显示：80/tcp -> 0.0.0.0:8080

# 3. 测试 localhost
curl http://localhost:8080

# 4. 检查 hosts 配置
cat /etc/hosts | grep web.local  # Linux
# 或在 Windows 查看 C:\Windows\System32\drivers\etc\hosts

# 5. 测试域名解析
ping web.local
```

### 问题 2：配置修改不生效

```bash
# 1. 测试配置语法
docker exec my-web nginx -t

# 2. 查看配置是否加载
docker exec my-web nginx -T | grep server_name

# 3. 重载配置
docker exec my-web nginx -s reload

# 4. 如果还不行，重启容器
docker restart my-web
```

### 问题 3：页面刷新 404

**原因：** Vue Router History 模式需要特殊配置

**解决：** 确保配置文件中有：
```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### 问题 4：静态资源 404

```bash
# 1. 检查挂载目录
docker exec my-web ls -la /usr/share/nginx/html/web_deploy/dist/

# 2. 检查文件权限
ls -la /home/blue/www/web_deploy/dist/

# 3. 确认 root 路径正确
docker exec my-web cat /etc/nginx/conf.d/vue-learning.conf | grep root
```

### 问题 5：WSL 中 Windows 浏览器无法访问

**解决方案：**

```bash
# 方法 1：使用 localhost（推荐）
# Windows 浏览器访问：http://localhost:8080

# 方法 2：获取 WSL IP
hostname -I
# 假设显示：172.20.10.5
# Windows 浏览器访问：http://172.20.10.5:8080

# 方法 3：Windows hosts 配置
# C:\Windows\System32\drivers\etc\hosts
127.0.0.1  web.local
```

## 💡 核心概念

### Bind Mount（绑定挂载）

```bash
-v /home/blue/www:/usr/share/nginx/html
```

**特点：**
- 直接映射宿主机目录到容器
- 宿主机文件修改，容器内立即生效
- 适合开发环境
- 路径必须是绝对路径

**语法：**
```
-v 宿主机路径:容器内路径[:选项]
```

**选项：**
- `ro`：只读挂载
- `rw`：读写挂载（默认）

### Nginx 配置加载机制

Nginx 主配置文件（`/etc/nginx/nginx.conf`）中包含：
```nginx
include /etc/nginx/conf.d/*.conf;
```

这意味着：
- `conf.d/` 目录下所有 `.conf` 文件会被自动加载
- 可以为每个项目创建独立配置文件
- 不需要修改主配置文件

### 端口映射

```bash
-p 8080:80
```

**格式：** `-p 宿主机端口:容器端口`

**说明：**
- 宿主机的 8080 端口映射到容器的 80 端口
- 访问 `localhost:8080` 实际访问容器的 80 端口
- 可以映射多个端口：`-p 8080:80 -p 8443:443`

## ⚠️ 注意事项

1. **路径必须使用绝对路径**
   ```bash
   # ❌ 错误
   -v ./www:/usr/share/nginx/html
   
   # ✅ 正确
   -v /home/blue/www:/usr/share/nginx/html
   ```

2. **配置文件必须以 .conf 结尾**
   ```bash
   # ✅ 会被加载
   vue-learning.conf
   
   # ❌ 不会被加载
   vue-learning.txt
   ```

3. **修改配置后必须重载**
   ```bash
   docker exec my-web nginx -s reload
   ```

4. **WSL 端口转发**
   - WSL2 自动转发 localhost 到 Windows
   - 使用 `localhost:8080` 即可在 Windows 访问
   - hosts 配置在 Windows 端进行

5. **文件权限问题**
   ```bash
   # 确保目录所有者正确
   sudo chown -R blue:blue /home/blue/nginx
   sudo chown -R blue:blue /home/blue/www
   ```

6. **容器重启后配置保留**
   - 使用挂载方式，配置文件在宿主机
   - 容器删除重建后，配置自动生效
   - 不需要重新配置

## 🎯 最佳实践

1. **统一管理配置**
   - 所有项目配置放在 `/home/blue/nginx/conf.d/`
   - 使用清晰的命名：`项目名.conf`

2. **使用只读挂载（生产环境）**
   ```bash
   -v /home/blue/www:/usr/share/nginx/html:ro
   -v /home/blue/nginx/conf.d:/etc/nginx/conf.d:ro
   ```

3. **配置自动重启**
   ```bash
   --restart unless-stopped
   ```

4. **定期备份配置**
   ```bash
   tar -czf nginx-config-backup.tar.gz /home/blue/nginx/
   ```

5. **使用 Docker Compose（可选）**
   ```yaml
   version: '3.8'
   services:
     web:
       image: nginx:alpine
       container_name: my-web
       ports:
         - "8080:80"
       volumes:
         - /home/blue/www:/usr/share/nginx/html:ro
         - /home/blue/nginx/conf.d:/etc/nginx/conf.d:ro
       restart: unless-stopped
   ```
