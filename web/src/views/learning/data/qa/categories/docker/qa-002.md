# Docker inspect 命令与容器挂载信息查看

## ❓ 问题

`docker inspect my-nginx | grep Mounts -A 10` 这个命令是什么意思？如何查看容器的挂载信息？

## ✅ 解答

### 命令详解

```bash
docker inspect my-nginx | grep Mounts -A 10
```

**命令分解：**

| 部分 | 说明 |
|------|------|
| `docker inspect my-nginx` | 查看容器的详细配置信息（JSON 格式） |
| `\|` | 管道符，将前面的输出传给后面的命令 |
| `grep Mounts` | 搜索包含 "Mounts" 的行 |
| `-A 10` | 显示匹配行及其后面的 10 行（After） |

### docker inspect 命令

`docker inspect` 返回容器或镜像的底层详细信息，包括：

- 网络配置
- 挂载信息
- 环境变量
- 启动命令
- 资源限制
- 状态信息

**基本用法：**
```bash
# 查看容器详细信息
docker inspect my-nginx

# 查看镜像详细信息
docker inspect nginx

# 查看多个容器
docker inspect container1 container2
```

### Mounts 挂载信息

**Mounts 包含的关键字段：**

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `Type` | 挂载类型 | `bind`、`volume`、`tmpfs` |
| `Source` | 主机路径 | `/home/user/html` |
| `Destination` | 容器内路径 | `/usr/share/nginx/html` |
| `Mode` | 挂载模式 | `rw`（读写）、`ro`（只读） |
| `RW` | 是否可读写 | `true` / `false` |
| `Propagation` | 挂载传播 | `rprivate`、`shared` |

**示例输出：**
```json
"Mounts": [
    {
        "Type": "bind",
        "Source": "/home/blue/html",
        "Destination": "/usr/share/nginx/html",
        "Mode": "",
        "RW": true,
        "Propagation": "rprivate"
    }
]
```

### 挂载类型说明

#### 1. bind（绑定挂载）
```bash
# 将主机目录挂载到容器
docker run -d -v /host/path:/container/path nginx
```
- 直接映射主机文件系统
- 主机路径必须存在
- 适合开发环境

#### 2. volume（数据卷）
```bash
# 使用 Docker 管理的数据卷
docker run -d -v my-volume:/container/path nginx
```
- 由 Docker 管理
- 数据持久化
- 适合生产环境

#### 3. tmpfs（临时文件系统）
```bash
# 挂载到内存
docker run -d --tmpfs /tmp nginx
```
- 存储在内存中
- 容器停止后数据丢失
- 适合临时数据

### grep 选项详解

```bash
# -A 10：显示匹配行及后面 10 行（After）
docker inspect my-nginx | grep Mounts -A 10

# -B 5：显示匹配行及前面 5 行（Before）
docker inspect my-nginx | grep Mounts -B 5

# -C 5：显示匹配行及前后各 5 行（Context）
docker inspect my-nginx | grep Mounts -C 5

# -i：忽略大小写
docker inspect my-nginx | grep -i mounts -A 10
```

### 更好的查看方式

#### 使用 jq 工具（推荐）

```bash
# 安装 jq（JSON 处理工具）
sudo apt install jq

# 只查看 Mounts 部分
docker inspect my-nginx | jq '.[0].Mounts'

# 查看特定字段
docker inspect my-nginx | jq '.[0].Mounts[0].Source'

# 格式化输出
docker inspect my-nginx | jq '.[0].Mounts[] | {Type, Source, Destination}'
```

#### 使用 --format 参数

```bash
# 直接提取 Mounts 信息
docker inspect --format='{{json .Mounts}}' my-nginx | jq

# 查看网络配置
docker inspect --format='{{json .NetworkSettings}}' my-nginx | jq

# 查看环境变量
docker inspect --format='{{.Config.Env}}' my-nginx
```

## 💡 实用技巧

### 查看容器的关键信息

```bash
# 查看容器 IP 地址
docker inspect --format='{{.NetworkSettings.IPAddress}}' my-nginx

# 查看容器状态
docker inspect --format='{{.State.Status}}' my-nginx

# 查看容器启动时间
docker inspect --format='{{.State.StartedAt}}' my-nginx

# 查看端口映射
docker inspect --format='{{json .NetworkSettings.Ports}}' my-nginx | jq
```

### 批量查看多个容器

```bash
# 查看所有运行容器的挂载信息
docker ps -q | xargs docker inspect --format='{{.Name}}: {{json .Mounts}}' | jq

# 查看所有容器的 IP 地址
docker ps -q | xargs docker inspect --format='{{.Name}}: {{.NetworkSettings.IPAddress}}'
```

## 🔗 相关命令

```bash
# 查看数据卷列表
docker volume ls

# 查看数据卷详细信息
docker volume inspect my-volume

# 查看容器的挂载点（简化版）
docker inspect -f '{{range .Mounts}}{{.Source}} -> {{.Destination}}{{println}}{{end}}' my-nginx

# 导出完整配置到文件
docker inspect my-nginx > container-config.json
```

## 📌 关键要点

1. **docker inspect** - 查看容器/镜像的完整配置信息
2. **Mounts** - 包含所有挂载信息（数据卷、绑定挂载等）
3. **grep -A** - 显示匹配行及后续行，方便查看 JSON 结构
4. **jq 工具** - 更优雅地处理 JSON 输出
5. **--format 参数** - 直接提取特定字段，避免使用 grep

## 🎯 常见场景

**场景1：检查数据是否正确挂载**
```bash
docker inspect my-nginx | grep -A 10 Mounts
# 确认 Source 和 Destination 路径是否正确
```

**场景2：排查容器数据丢失问题**
```bash
docker inspect my-nginx | jq '.[0].Mounts'
# 检查挂载类型，确认是否使用了持久化存储
```

**场景3：迁移容器配置**
```bash
docker inspect my-nginx > backup.json
# 保存配置，用于在其他主机重建容器
```
